# 产品需求文档：浣熊卡路里（RaccoonCal）

## 1. 产品背景与定位

### 1.1 市场机会
- 全球健康意识上升，卡路里追踪应用市场巨大，但现有产品（如MyFitnessPal）偏工具化，用户容易因数据压力而流失
- 2025年乌克兰团队Reface推出的BitePal，通过AI识别+游戏化角色（小浣熊）快速获得市场，证明**"轻松有趣"的饮食管理**是用户刚需
- 国内及新兴市场缺乏类似产品，存在空白机会

### 1.2 产品定位
**浣熊卡路里**：一款面向年轻用户（18-35岁）的轻量级饮食记录应用，以AI拍照识别为核心，通过养成系游戏化体验降低用户记录负担，让健康管理变得有趣。

### 1.3 核心价值
- **极简记录**：拍照即得卡路里，无需手动输入
- **情感陪伴**：虚拟宠物（浣熊）以幽默方式反馈饮食，消解焦虑
- **正向激励**：游戏化任务鼓励健康行为，而非冷冰冰的数据警告

## 2. 目标用户

### 2.1 核心用户画像

| 属性 | 描述 |
|------|------|
| **年龄** | 18-30岁，学生或职场新人 |
| **性别** | 女性占60%，男性40% |
| **饮食管理习惯** | 偶尔关注健康，但讨厌复杂记录；容易被可爱、有趣的事物吸引 |
| **典型场景** | 日常三餐、下午茶、朋友聚餐时拍照记录；希望了解食物热量又不想太麻烦 |
| **痛点** | 现有App太枯燥，坚持不下去；对卡路里数字无感，需要更直观的反馈 |

### 2.2 次要用户
- 健身新手：需要简单工具辅助饮食控制
- 社交媒体活跃用户：愿意分享虚拟宠物成就

## 3. 核心功能

### 3.1 拍照识别卡路里（MVP核心）
- **功能描述**：用户拍摄食物照片，应用自动识别食物种类并估算卡路里，显示在浣熊的对话框内
- **交互流程**：
  1. 点击底部相机按钮
  2. 拍照或从相册选择
  3. 系统识别并返回结果（3秒内）
  4. 浣熊角色以幽默口吻点评（如"这盘炸鸡的热量足够我跑三圈啦！不过偶尔放纵没关系～"）
  5. 用户可手动调整食物份量或种类

### 3.2 虚拟宠物养成（游戏化核心）
- **角色设定**：一只名叫"小浣熊"的虚拟宠物，会随着用户饮食记录而成长、换装、互动
- **核心机制**：
  - **饮食打卡**：每记录一次食物，浣熊获得经验值，连续打卡可解锁新表情/动作
  - **营养评分**：系统根据饮食均衡度（蛋白质/碳水/脂肪比例）给浣熊"活力值"，高分可让浣熊开心跳舞
  - **挑战任务**：每日任务（如"今天吃够5种颜色食物"）、成就系统（如"连续一周记录"）
  - **装扮系统**：用户可用记录获得的积分兑换浣熊配饰（帽子、眼镜等）

### 3.3 轻量社交元素
- **好友榜**：与微信好友或App内好友PK浣熊成长等级（非卡路里数据，避免身材焦虑）
- **分享卡片**：生成浣熊趣味点评截图，分享到社交平台吸引新用户

### 3.4 个人数据中心
- 每日/每周饮食报告，用浣熊视角总结（如"这周你带我吃了好多蔬菜，我变强壮啦！"）
- 简单趋势图（卡路里摄入趋势）

## 4. 用户体验与交互设计

### 4.1 用户旅程
1. **首次打开**：
   - 引导页：展示浣熊角色，承诺"拍照即得，轻松有趣"
   - 快速注册：手机号/微信一键登录
   - 领取初始浣熊（默认形象）

2. **日常使用**：
   - 打开App → 浣熊在主页挥手问候 → 点击相机拍照 → 识别结果弹出浣熊点评 → 用户确认/修改 → 浣熊获得经验 → 跳回主页

3. **游戏化互动**：
   - 点击浣熊进入养成页，查看等级、任务、装扮
   - 每日签到奖励

### 4.2 界面设计原则
- **温暖可爱**：主色调采用暖黄+森林绿，图标圆润
- **轻量化**：首页只放相机、浣熊、任务三个入口
- **情感化文案**：所有提示都用浣熊口吻（例如"主人，今天想吃啥？"）

## 5. 变现模式

初期以积累用户为主，后期逐步引入：

- **订阅会员**：
  - 基础功能免费（拍照识别、基础浣熊养成）
  - 会员可解锁高级装扮、详细营养分析、无限次历史记录
- **广告**：非侵入式激励视频广告，用户观看后可获得积分兑换装扮
- **品牌合作**：与健康食品品牌合作推出限定装扮（如"XX酸奶联名帽子"）

## 6. 关键指标与验证

### 6.1 MVP阶段核心指标
- **次日留存率** > 40%（证明游戏化对用户的吸引力）
- **周活跃用户日均记录次数** > 2次（核心功能使用频率）
- **用户主动分享率** > 15%（验证社交传播潜力）

### 6.2 数据追踪
- 记录用户对识别结果的修改频率，评估API准确性需求
- 追踪任务完成率，调整游戏化难度

## 7. 开发路线图（3个月MVP）

### 第1个月：核心功能开发
- 后端环境搭建，集成LogMeal API
- 前端实现拍照、识别展示、手动修改
- 基础浣熊角色（静态图像+简单弹窗文案）

### 第2个月：游戏化系统
- 浣熊动画（Lottie）集成
- 任务系统后端+前端展示
- 装扮商店雏形（积分兑换）

### 第3个月：社交与优化
- 好友排行榜开发
- 分享卡片生成
- 用户测试与Bug修复，准备上线

## 8. 风险与对策

| 风险 | 对策 |
|------|------|
| API识别准确率低导致用户流失 | 允许手动调整，并收集用户反馈数据，后期训练自有模型 |
| 游戏化新鲜感消退 | 定期更新节日装扮、限时任务 |
| 同类产品抄袭 | 快速迭代，积累早期用户口碑，建立社区 |
